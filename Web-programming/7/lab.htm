1 Представление ссылок на веб-странице

Ссылки – это выделенные объекты на веб-странице (слова, изображения и др.), которые при наведении курсора мыши меняют свои параметры с появлением на экране изображения "ладошки". После щелчка мыши по ссылке происходит поиск и загрузка требуемой веб-страницы.
В HTML 4.01 тег <a> мог быть как ссылкой (англ. hyperlink), так и якорем (англ. anchor). В HTML5 тег <a> всегда является ссылкой, но если он не имеет атрибута href, он представляет собой место (англ. placeholder) для ссылки.

Оглавление

1.1 Задание атрибутов ссылок

Для задания ссылки и ее атрибутов используется элемент <a> ссылка </a>, который содержит такие атрибуты:

  • charset – задает кодировку символов связываемого документа. Не поддерживается HTML5;
  • coords – указывает координаты ссылки. Не поддерживается HTML5;
  • download – указывает на то, что указанный ресурс, заданный атрибутом href, будет загружен в компьютер при щелчке по ссылке. Атрибут добавлен в HTML5. В качестве значения атрибута указывается новое имя загружаемого файла. Ограничений на возможные значения не имеется и браузер, определив содержимое, автоматически добавит расширения (.img, .pdf, .txt, .html и др.) к имени файла. Если значение атрибута не задано, принимается исходное имя файла. Атрибут поддерживается браузерами Chrome 14.0, Firefox 20.0 и Opera 15,0. Не поддерживают – Internet Explorer и Safari;
  • href – указывает URL-адрес Web-страницы, которая будет загружена после щелчка мыши по ссылке (если URL-адрес указывает на видеофайл или звуковой файл, то начнется воспроизведение соответственно видеофрагмента или звука);
  • hreflang – указывает язык подключаемого документа;
  • media – указывает, для какого медиа устройства оптимизированы ресурсы (см. подраздел 3.2 лаб. раб. №2). Атрибут добавлен в HTML5;
  • name – указывает имя в документе, куда будет перемещен указатель при внутреннем использовании ссылок. Не поддерживается HTML5. Вместо атрибута name необходимо использовать глобальный атрибут id (см. подразделы 1.5 данной лаб. раб. и 2.3 лаб. раб. №1);
  • rel – указывает отношение между текущим и связываемым документами. Используется, если задан атрибут href. Отметим, что атрибут rel браузерами не обрабатывается, но может быть использован поисковыми системами, которые благодаря ему получат больше информации о ссылке. Может принимать следующие значения:
    • alternate – ссылается на альтернативную версию документа, например, печатная страница, перевод или копия;
    • author – ссылается на автора документа;
    • bookmark – указывает URL, используемый для закладки;
    • help – ссылается на документ, предоставляющий помощь;
    • license – ссылается на информацию о лицензировании документа;
    • next – указывает следующий отмеченный документ;
    • nofollow – ссылается на недоступный документ, например, платную ссылку;
    • prefetch – указывает на то, что браузер не должен посылать HTTP заголовок перенаправления пользователя;
    • prefetch – указывает, что целевой документ будет кэширован;
    • prev – указывает предыдущий отмеченный документ;
    • search – ссылается на поисковые средства (инструменты) для документа;
    • tag – тег (ключевое слово) для текущего документа;

    Оглавление

  • rev – указывает соотношение между подключаемым документом и данным документом. Не поддерживается HTML5;
  • shape – используется вместе с атрибутом coords и указывает размер, форму и место ссылки в элементах <object> или <img>:
    • default – указывает всю область;
    • rect – определяет прямоугольную область;
    • circle – определяет круглую область;
    • poly – определяет область многоугольника.
    Не поддерживается HTML5. Укажем, что для создания карты изображения, которая работает во всех браузерах, необходимо использовать элемент <area>;
  • target – указывает место (окно или фрейм), где будет открыт полученный по ссылке документ:
    • _blank – документ открывается в новом окне;
    • _self – документ открывается в текущем окне (значение по умолчанию);
    • _parent – документ открывается в окне родительского элемента;
    • _top – документ открывается во весь экран;
    • framename – документ открывается в окне (фрейме) с указанным именем;
  • type – указывает MIME-тип подключаемого документа.

Отметим, что атрибуты download, hreflang, media, rel, target и type нельзя использовать, если не указан атрибут href.
Для задания ссылки на веб-страницу с описанием лабораторной работы №1 можно применить следующий тег:

<a href="../LAB1/lab.htm"> лаб. раб. №1</a>,

где "../LAB1/lab.htm" – задание пути к файлу "lab.htm" на данном компьютере, которое поддерживают все ведущие браузеры.
Для изменения цвета ссылки в HTML 4.01 использовались следующие атрибуты тега <body>:

  • link – задает цвет не просмотренной ссылки (по умолчанию синий);
  • vlink – задает цвет посещенной ссылки (по умолчанию сиреневый);
  • alink – задает цвет ссылки после щелчка мыши по ней, но до загрузки Web-страницы (по умолчанию красноватый).

Атрибуты link, vlink и alink не поддерживаются HTML5. Для изменения цвета ссылок и других их свойств необходимо использовать псевдоклассы CSS (см. подраздел 1.3 ).
Ссылка может быть задана также в виде изображения. Для этого необходимо тег, задающий рисунок (<img>), поместить внутри элемента <a> … </a>:

<a href="../LAB2/lab.htm"><img src="fish.gif"></a>

В примере №1 приведено использование стандартных ссылок, а также ссылок в виде изображения.

1.2 Использование тега <base>

В качестве URL-адреса ссылки может быть задан как абсолютный адрес, который включает базовый адрес (URL-адрес веб-страницы) и относительный адрес (путь к файлу внутри страницы), так и только относительный адрес.
Элемент <base> с помощью своих атрибутов href и target позволяет устанавливать значения по умолчанию для ссылок документа:

  • href – задает базовый URL-адрес для всех ссылок, имеющих относительные URL-адреса;
  • target – указывает место (окно или фрейм), где будет открыт полученный по ссылке документ (имеет те же значения, что и атрибут target элемента <a>).

1.3 Использование CSS в ссылках

Каскадные листы стилей, помимо псевдоклассов, описанных в подразделе 1.2 лаб. раб. №2, имеют следующие псевдоклассы, задающие свойства ссылок:

  • a:link – для задания параметров (цвета, формы и др.) тех ссылок, которые еще не были использованы для загрузки веб-страниц;
  • a:visited – для задания параметров тех ссылок, которые уже были использованы;
  • a:active – для задания параметров активных ссылок, т.е. тех ссылок, которые выделены, например, нажатием левой кнопки мышки.

Отметим, что псевдо-классы описываются так же, как и классы (см. раздел 1, а также подразделы 3.1 и 3.2 лаб. раб. №2).
Если на веб-странице нужно использовать несколько ссылок, имеющих разные свойства, можно вместе с псевдоклассами задавать обычные классы. Например, можно создать класс изображений ссылок в виде кнопок. Ниже приводятся каскадные листы стилей, которые описывают класс ссылок a.kn, задающий для кнопки тип и размер шрифта, а также свойства границы:

/* ссылка в виде кнопки */
a.kn {
font-style:italic;
font-size:0.6cm;
border:outset blue 6px;
text-decoration:none.
} Оглавление

Кроме того, для четырех состояний ссылок: link, visited, active и hover ( см. подраздел 1.2 лаб. раб. №2) заданы четыре псевдо-класса, указывающие цвет надписи и фона кнопки для каждого состояния ссылки, соответственно:

/* для непосещенной ссылки */
a.kn:link {
color:yellow;
background-color:#C0C0C0;
}
/* для посещенной ссылки */
a.kn:visited {
color:#90090;
background:#00E0E0;
}
/* для отмеченной ссылки */
a.kn:active {
color:#0000FF;
background:#FFFF00;
}
/* для ссылки с наведенным курсором мышки */
a.kn:hover {
color:white;
background:#FF60A0;
} .

Эти описания параметров ссылок добавлены во внешний файл CSS "my_d.css" и используются в примере №1 для задания свойств ссылок с помощью каскадных листов стилей.
Ссылка класса "kn" задается так:

<a class="kn" … >.

1.4 Изображение курсора на веб-странице

При наведении курсора мыши на ссылку обычно появляется изображение "ладошки". В CSS имеется возможность изменить вид курсора (и не только для ссылок, но и для других элементов веб-страницы, например, изображений). Для этого необходимо использовать свойство cursor, которое устанавливает форму курсора, когда он находится в пределах элемента. Вид курсора зависит от операционной системы и заданных значений:

  •     alias – курсор указывает на создание псевдонима (англ. alias);
  •      all-scroll – курсор указывает на скролирование в разных направлениях;
  •      auto – вид курсора по умолчанию для текущего элемента;
  •      cell – курсор указывает на выделение ячейки (или группы ячеек);
  •      context-menu – курсор указывает, что контекстное меню доступно;
  •      col-resize – курсор указывает, что размер колонки может быть изменен по горизонтали;
  •      copy – курсор указывает на копирование;
  •      crosshair – курсор указывается в виде креста;
  •      default – изображение курсора по умолчанию;
  •      e-resize – курсор указывает, что край прямоугольного объекта (box) передвигается вправо (на восток) (не отличается от cursor:ew-resize – см. ниже);
  •      ew-resize – курсор указывает на двунаправленное передвижение края объекта box (вправо-влево, восток-запад);
  •      grab – курсор указывает на перетаскивание объекта;
  •      help – курсор указывает на доступность помощи;
  •      move – курсор указывает на перемещение объекта;
  •      n-resize – курсор указывает, что край объекта box передвигается вверх (на север) (не отличается от cursor:ns-resize – см. ниже);
  •      ne-resize – курсор указывает, что край объекта box передвигается вверх и вправо (на северо-восток) (сейчас не отличается от cursor:nesw-resize – см. ниже);
  •      nesw-resize – курсор указывает на двунаправленное передвижение края объекта box (вверх/вправо и вниз/влево, северо-восток и юго-запад);
  •      ns-resize – курсор указывает на двунаправленное передвижение края объекта box вверх-вниз (север-юг);
  •      nw-resize – курсор указывает, что край объекта box передвигается вверх и влево (на северо-запад) (сейчас не отличается от cursor:nwse-resize – см. ниже);
  •      nwse-resize – курсор указывает на двунаправленное передвижение края объекта box (вверх/влево и вниз/вправо, северо-запад и юго-восток);
  •      no-drop – курсор указывает, что перетаскиваемый элемент не может быть здесь оставлен;
  •      none – для элемента курсор не будет выведен;
  •      not-allowed – курсор указывает, что запрашиваемое действие не будет выполнено;
  •      pointer – курсор указывает на ссылку;
  •      progress – курсор указывает, что программа занята;
  •      row-resize – курсор указывает, что размер строки может быть изменен по вертикали;
  •      s-resize – курсор указывает, что край объекта box передвигается вниз (на юг) (сейчас не отличается от cursor:ns-resize – см. выше);
  •      sw-resize – курсор указывает, что край объекта box передвигается вниз и влево (на юго-запад) (не отличается от cursor:nesw-resize – см. выше);
  •       text – курсор указывает, что текст может быть выделен;
  •      url(../tre1.gif), auto – значением является список разделенных запятыми URL-адресов курсоров пользователя. Последним в списке нужно указать стандартное изображение курсора. Оно будет использовано, если ни один из курсоров пользователя не будет доступен, например: cursor:url(back.gif), url(tre1.gif), auto;
  •       vertical-text – курсор указывает, что вертикальный текст может быть выделен;
  •       w-resize – курсор указывает, что край прямоугольного объекта (box) передвигается влево (на запад) (не отличается от cursor:ew-resize – см. выше);
  •       wait – курсор показывает, что программа занята;
  •       zoom-in – курсор указывает на уменьшение размера;
  •       zoom-out – курсор указывает на увеличение размера;
  •       initial – устанавливает для свойства значение по умолчанию;
  •       inherit – свойство наследует значение свойства родительского элемента.

Укажем, что изображение соответствующего курсора можно увидеть, установив в поле левее его значения курсор мышки. В примере №1 показано использование изображение курсора в виде знака вопроса.

1.5 Использование внутренних ссылок

Внутренние ссылки используются для облегчения доступа к отдельным частям документа, представленного на веб-странице. Для этого в HTML-документе с помощью тега <a> в требуемых местах документа устанавливаются метки доступа. При этом в теге <a> вместо атрибута href используется атрибут id, задающий имя метки доступа. К этому месту на веб-странице можно обратиться с помощью обычной ссылки, в которой в качестве значения атрибута href указывается имя метки с предшествующим знаком #.
Внутренние ссылки можно также использовать для доступа к внутренним частям другого документа. В этом случае к URL-адресу документа добавляется знак # и имя внутренней ссылки в этом документе. Например, с помощью ссылки

<a href="../LAB2/lab.htm#2">

обеспечивается доступ не к началу описания лабораторной работы №2, а к ее разделу 2.
В примере №1 показано применение внутренних ссылок для текущего и внешнего документов.

1.6 Сегментированная графика

К сегментированной графике относятся изображения, условно разбитые на отдельные части в виде определенных геометрических фигур, каждая из которых представляет собой область чувствительности к щелчку мыши и является ссылкой.
Для перечисления областей чувствительности используется элемент <map>, атрибут которого name задает имя карты с описанием этих областей.
Каждая область чувствительности описывается с помощью одиночного тега <area>, который содержит такие атрибуты:

  • shape – задает геометрическую фигуру, описывающую форму области чувствительности:
    • rect – прямоугольник;
    • circ – окружность;
    • poly – многоугольник.
  • coords – задает координаты этих фигур:
    • для прямоугольника – в виде координат его левого правого угла и левого нижнего (x1,y1,x2,y2);
    • для окружности – в виде пары координат центра и величины радиуса (x,y,r);
    • для многоугольника – в виде пар координат для каждой вершины (x1,y1,x2,y2, …).
  • href – задает URL-адрес требуемой веб-страницы;
  • nohref – указывает на то, что данная область нечувствительна к наведению мышки (должна быть объявлена первой);
  • alt – задает поясняющую надпись под ссылкой, которая появляется некоторое время спустя после наведения указателя мышки на ссылку (аналогично атрибуту title тега <a>).

Например, для задания фигур и координат трех областей чувствительности (они выделены цветом) изображения <img src="seg.bmp" align="middle" border="0"> используются следующие значения атрибутов shape и coords:

  • для прямоугольника – shape=rect coords=10,6,50,30
  • для треугольника – shape=poly coords=10,40,10,80,50,40
  • для окружности – shape=circ coords=88,49,20

Чтобы связать между собой рисунок с сегментированной графикой, заданный тегом <img> с картой изображения, заданной контейнером <map> … </map>, необходимо указать в теге <img> атрибут usemap, значением которого является значение параметра name тега <map> с добавленным первым символом решетки (#).
В примере №1 показано использование разных видов ссылок на Web-странице.
